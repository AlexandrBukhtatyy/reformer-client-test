# Задача

Спланируй работу по созданию формы на основе спецификации spec/insurance-application-form.md.
План сохрани в md-файле (PLAN.md).
Генерировать код не надо.

# Требования

Используй @reformer/ui для:

1. создания пошаговых форм (form-navigation)
2. создания масивов форм (form-array)

Используй MCP сервер reformer для:

1. получения документации (reformer://docs и reformer://api)
2. примеров кода (reformer://examples)
3. решения возникших проблем (reformer://troubleshooting)
4. регистрации проблем возникших при работе (report_issue)
5. структура кода (reformer://docs)

Форма должна:

- Быть пошаговой (6 шагов)
- Иметь отдельные файлы валидации для каждого шага
- Использовать useStepForm для навигации
- Поддерживать вычисляемые поля (endDate, age, experience)
- Иметь условную видимость полей
- Используй TailwindCSS для верстки формы (установленной версии)

## Обязательно: Фиксация всех правок

После первоначальной генерации формы, КАЖДЫЙ РАЗ когда вносишь исправления или доработки - вызывай `report_issue`.

**Когда вызывать:**

- Исправление ошибки типизации
- Исправление runtime ошибки
- Доработка после ревью/тестирования
- Любое изменение сгенерированного кода

**Обязательные теги:**

- `branch:<name>` - текущая git ветка (например `branch:feature/insurance-form`)
- `file:<path>` - файл где была ошибка (например `file:schemas/step1-validation.ts`)

**Опционально в context:**

- `examples` - код до/после исправления
- `relatedFiles` - связанные файлы
- `notes` - почему так произошло
